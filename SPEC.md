# SPEC.md — myapt

## Project
`myapt`

## One-line
내 연봉, 현금, 대출 조건, 원하는 지역을 입력하면 대한민국 아파트 매매 실거래가 기준으로 구매 가능한 `단지 + 전용면적 타입`을 보여주는 한국어 웹앱.

## PM scope
MVP는 아래를 반드시 포함한다.
- 한국어 웹앱
- 재무 입력 → 예산 계산 → 실거래가 기반 검색까지 한 흐름
- 결과 단위는 `단지 + 전용면적 타입`
- 가격 기준은 공식 아파트 매매 실거래가
- 네이버 부동산 링크 연결
- 규칙 엔진 기반 대출 추정
- DB 기반 검색
- 관리자/설정 최소 버전
- 로컬 실행 가능
- 문서/환경변수/테스트 포함

## User stories
1. 사용자는 나이, 소득, 현금, 기존부채, 대출 옵션을 입력해 내 최대 구매 가능 예산을 알고 싶다.
2. 사용자는 지역/세대수/연식/전용면적/최근 거래 기간으로 필터링해 실제 구매 가능한 아파트를 찾고 싶다.
3. 사용자는 결과에서 왜 가능/불가능한지, 어떤 규칙이 적용됐는지 보고 싶다.
4. 사용자는 결과 클릭으로 네이버 부동산으로 이동하고 싶다.
5. 관리자는 규칙 버전, 금리 가정, 취득비용 가정, 데이터 동기화 상태를 확인하고 싶다.

## Acceptance criteria
- 재무 입력 시 구매 가능 최대가격, 최대 대출액, 필요 자기자본, 월 상환액, 적용 규칙 요약이 계산된다.
- 검색 결과는 실거래가 기반이며 `단지 + 전용면적 타입` 단위다.
- 필터: 지역, 세대수, 연식, 전용면적, 최근 거래 기간, 정렬.
- 결과에 단지명, 전용면적, 주소/지역, 세대수, 준공연도, 대표 실거래가, 최근 거래일, 거래건수, 예산 차이, 필요 대출액, 월 상환액, 네이버 링크가 보인다.
- 규칙 엔진은 DB/설정 기반으로 바꿀 수 있다.
- 공공 API 직접 호출은 서버 수집용으로만 사용되고 사용자 검색은 DB 기반이다.
- 최소 단위테스트와 핵심 플로우 테스트가 있다.

## Non-goals
- 회원가입/로그인
- 지도 중심 UI
- 전세/월세/빌라/오피스텔
- 실시간 은행 심사 결과
- 고도화된 추천 알고리즘

## Product assumptions
- MVP의 데이터 수집은 공식 공공데이터 API + 개발용 mock seed를 함께 지원한다.
- 규칙 엔진은 현재 한국 주담대 규칙의 단순화된 실무형 추정 모델로 구현한다.
- 네이버 링크는 direct link가 없으면 검색 링크 fallback을 사용한다.
